# Docs

This repo keeps lightweight project documentation in a small `docs/` tree.

## What Goes Where

- `docs/specs/`: product and feature requirements
- `docs/architecture/`: implementation-oriented design notes
- `docs/runbooks/`: operational procedures and recovery steps

## How To Add A New Doc

1. Put product requirements in `docs/specs/`.
2. Put implementation details and data flow in `docs/architecture/`.
3. Add the new file to this index so future work is discoverable.
